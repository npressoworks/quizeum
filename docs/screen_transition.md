# quizeum 画面遷移図・UI設計仕様書

本ドキュメントは、クイズ投稿SNS「quizeum」におけるユーザーの操作フローおよび各画面間の遷移仕様を定義します。本仕様は、要件定義書に定義されたソーシャル機能（フォロー、ブックマーク、いいね/リアクション、タグ、ジャンル）を含むすべての機能要件を満たす画面群を完全に網羅しています。

## 2. 各画面の機能仕様および遷移ルール

### 2.1 ホーム画面 (Page: `/`)
* **機能概要**:
  * 全ユーザー（ゲスト含む）が最初にアクセスするサービスのエントランス。
  * 「新着順」「人気順（プレイ数）」「トレンド順（ブックマーク数）」のクイズカードをタブで切り替えて表示。
  * ログイン済ユーザーの場合は、自分がフォローしているユーザーの最新投稿を表示する「タイムライン」タブも追加。
  * 主要ジャンル（プログラミング、歴史、雑学、語学など）のアイコンナビゲーションを設置。
  * ジャンル、難易度（1〜10の数値評価または範囲指定）、問題数、未プレイ/プレイ済み（ログイン時のみ）を絞り込める「複合検索フィルタ」を設置。
* **遷移ルール**:
  * 任意のクイズカードをクリックすると、対象の「クイズ詳細画面」へ遷移。
  * 任意のクイズリストカードをクリックすると、「クイズリスト詳細画面」へ遷移。
  * ジャンルアイコンをクリックすると、「ジャンル別クイズ一覧画面」へ遷移。
  * 総合リーダーボードリンクから「総合リーダーボード画面」へ遷移。
  * 未ログイン状態でブックマーク登録やフォローなどのアクションを実行しようとした場合、自動的に「認証画面」へポップアップまたは遷移。

### 2.2 認証画面 (Page: `/login`)
* **機能概要**:
  * Firebase Authを使用したメールアドレス/パスワード認証、およびGoogle等によるソーシャルサインインを提供。
  * 「ログイン」と「新規アカウント登録」のフォームをタブやボタンでスムーズに切り替え可能。
  * **E2Eテスト用モックログインボタンの露出制限**: ローカルでの開発およびE2Eテスト自動化のために用意されたモックログインボタン（ワンクリック自動認証ボタン等）は、本番環境（`process.env.NODE_ENV === 'production'`）においては画面上およびDOMから物理的・完全に非表示・除外され、悪意ある第三者による認証迂回および特権セッション乗っ取りを徹底的にブロックする。
* **遷移ルール**:
  * サインインまたはサインアップ完了後、元の画面（リダイレクト元）または「ホーム画面」に自動遷移。

### 2.3 クイズ詳細画面 (Page: `/quiz/[id]`)
* **機能概要**:
  * 対象クイズのメタ情報（タイトル、説明、問題数、難易度（1〜10の10段階数値評価）、ジャンル、タグ）および作成者プロフィールを表示。
  * 登録されている「タグ（ハッシュタグ形式）」を表示し、クリック可能に設計。
  * 「歴代ハイスコア（上位5名）」および「最速全問正解タイム（タイムアタック）」のリーダーボードを表示。
  * **良問バッジ表示**: `reviewBadge` フィールドに基づく「圧倒的良問」「高評価良問」「賛否両論」等の評価バッジと `reviewScore`（良問率）を表示する。評価数が10件未満の場合は表示なし。
  * ブックマーク登録/解除ボタン（ログイン中のみアトミックにトグル可能）。
  * プレイ開始UIにて、以下の3つの「プレイモード」を選択可能。
    1. **通常モード**: 1問解くごとに即座に正誤判定と解説を表示。
    2. **模擬試験モード**: 1問ずつの解説を挟まず、全問解いた後に結果を一括表示・一括採点。
    3. **フラッシュカードモード**: 回答の入力を挟まず、タップで「問題（表）」と「解説（裏）」を切り替える高速暗記学習。
* **遷移ルール**:
  * プレイ開始ボタンクリックにより、選択したモードで「クイズプレイ画面」へ遷移。
  * タグをクリックすると、「タグ別クイズ一覧画面」へ遷移。
  * 作者のアイコン・表示名クリックで「プロフィール画面」へ遷移。

### 2.4 クイズプレイ画面 (Page: `/quiz/[id]/play`)
* **機能概要**:
  * クイズの解答を行うインタラクティブなClient-side画面。
  * **設問ブックマーク・マイリスト追加UI**: 各設問の表示カード（ウミガメのスープではチャット上部）に「設問をブックマークする」トグルボタンと、「マイリストに追加」アクションボタンが設置され、解答中であってもその設問単体をお気に入り登録や自分の問題集リストに追加可能。
  * **制限時間タイマー**: 設定されている場合、カウントダウンを表示。0秒になると自動で回答を締め切り不正解として処理。模擬試験プレイ時は、設問ごとのタイマーは作動せず「クイズ全体の総合制限時間」が適用され、時間内であれば画面のナビゲーションや「戻る」ボタンを用いて前の設問への移動・解答の見直しが自由に行える。
  * **ヒント機能**: 「ヒントを表示」ボタンをクリックすると、設定されたヒントテキストをポップアップまたはカード下部に表示。
  * **中断保護（セッション維持）**: プレイ中にブラウザリロードやネットワーク一時切断が発生しても、`localStorage` から解答セッション（解答済みの設問とスコア）を自動復元し、途中から再開可能にする。オフラインのままプレイを完了した場合は、ローカルの正誤情報のみを表示する「オフライン用結果画面」へ遷移する。ただし、クイズリスト（問題集）のプレイ中にオフライン状態になり、結果画面から「次のクイズに進む」をクリックした場合は、遷移をブロックしてオフラインエラー（トーストまたはダイアログ）を表示し、接続復旧を促す。
  * **選択式クイズ UI (`multiple-choice` / `true-false`)**:
    - 設問に登録された正解の数に応じて入力UIを切り替える。正解が **1 件** のときは各選択肢を **ラジオボタン**、**2 件以上** のときは **チェックボックス**（複数選択）で表示する。
    - プレイヤーは選択肢を選んだあと、**「解答を確定する」** ボタンを押して初めて解答が送信・正誤判定される（選択肢クリックのみでは送信しない）。並び替えクイズと同様の「選択 → 確定」の2段階操作とする。
    - 複数正解設問の正誤判定は、登録された正解選択肢の集合と、プレイヤーが選んだ選択肢の集合が完全一致した場合のみ正解とする。解答データは選択した Choice の `id` をカンマ区切りで `questionAnswers` に保存する（単一正解の旧形式は id 1 件のみ）。
  * **並び替えクイズ UI (sorting)**: 
    - 提示された複数の選択肢要素（2〜6個）がシャッフルされた状態で表示されます。
    - 各要素の左にドラッグハンドル（グリップアイコン）を配置し、**ドラッグ＆ドロップのみ**で順序を入れ替えます（上下ボタン ▲/▼ による順序変更UIは使用しません）。実装は `@dnd-kit/core` を推奨し、ドラッグ中・ドロップ時のリスト再配置アニメーションで操作感を高めます。
    - タッチ端末では誤スクロールを避けるため、短いドラッグ閾値後にソートを開始します。キーボードフォーカス時は矢印キー等で並べ替え可能とし、アクセシビリティ要件を満たします。
    - 並び替え完了後、「解答を確定する」ボタンをクリックして確定・正誤判定へ進みます（確定時に並べ替え後の要素ID列を解答として送信）。
  * **連想クイズ UI (association)**:
    - 初期表示時は「ヒント1」のみを表示。
    - 「次のヒントを表示（残りN件）」ボタンをクリックすることで、段階的に次のヒントが追加でスライドイン/フェードイン表示されます（最大5つ）。
    - プレイヤーはヒントが何件開示されている状態であっても、任意のタイミングで下部のテキスト入力フィールドから解答を送信して挑戦できます。
  * 問題画像がある場合は、問題文の上部または側部にレスポンシブなサイズで表示。
  * **ウミガメのスープ（AI対話チャット）**: 出題タイプが `lateral-thinking` の場合、プレイ画面は **2カラムレイアウト** に切り替わり、以下の仕様でAI対話UIを提供する。
    - **左カラム（チャットエリア）**: プレイヤーが自由記述（最大100文字）で質問を入力し、AIの判定結果と補足コメントをチャットバブル形式で時系列に表示する。AIは過去の対話履歴（直近最大20回分）を参照するステートフル対話で判定を行い、完全一致の質問は既存回答をキャッシュして返す（キャッシュ時は「📋 既存の回答」バッジ表示）。
    - **右カラム（Q&Aリストパネル）**: これまでの質問（Q）と回答（A）のペアをリスト形式で常時表示し、一目で確認可能。スクロール対応。
    - **ターン制限インジケーター**: 無料ユーザーのみ入力欄の下に「本日の残り質問数: N/20（同一クイズに対して1日最大20回制限）」を表示。有料プランユーザーには表示しない。毎日深夜0:00にリセット。
    - 解答を確信した時点で「真相を解き明かす」ボタンから最終回答を入力。提出された回答は**B2ハイブリッド真相判定**（必須キーワードがすべて含まれていればAIをバイパスして即合格、不足時のみAI判定にフォールバック）によって判定され、合格（正解）するとプレイ完了。
    - **サーバーサイド認証・認可署名検証**: AI質問（`/api/attempt/ask-ai`）および真相判定（`/api/attempt/verify-truth`）のリクエスト送信時には、必ずヘッダーに `Authorization: Bearer <ID_TOKEN>` を付与し、サーバー側で JWT 署名検証と本人確認（`attempts.userId == JWT.uid`）を行い、他人の履歴書き換えを防止する。またプレミアムプランのターン制限解除においては、サーバー側で `users/{uid}` ドキュメントの最新情報を引き直して厳格にプラン検証（特権昇格防止）を行う。
* **遷移ルール**:
  * すべての設問に解答し終えた瞬間（模擬試験モードの場合は一括採点実行時、ウミガメのスープでは真相判定合格時）に、「クイズ結果画面」へ自動遷移。
  * 途中で離脱する（ホームや戻るボタン等）場合は、現在のセッション破棄に対する警告ダイアログを表示。
  * **ゲストプレイアクセス制限**: 出題タイプが `lateral-thinking`（ウミガメのスープ）のクイズにおいて、未ログイン（ゲスト）の状態で直接アクセス、または詳細画面から遷移しようとした場合、自動的に「認証画面（`/login`）」へとリダイレクトされます（ログイン完了後に元のプレイ画面へ自動遷移）。


### 2.5 クイズ結果画面 (Page: `/quiz/[id]/result`)
* **機能概要**:
  * 最終結果の表示（正解数/総問題数、正解率、タイムアタックモードの場合は経過解答秒数）。水平思考形式（ウミガメのスープ）では、クリア時間（秒数）がリーダーボードに記録される。
  * 全問の正誤状況リストと、各設問の**「あなたの回答」および「正解」**・詳細解説（リッチマークダウン、外部リンク対応）を一覧で確認。回答・正解は問題形式に応じて整形表示される（選択式：選択肢テキスト、並び替え：「要素A → 要素B → 要素C」形式、フラッシュカード：「覚えていた / 覚えていなかった」等）。旧レコードで回答データがない場合は「（記録なし）」と表示する。各設問の解説エリアには、個別に「ブックマーク」や「マイリストに追加・削除」ができるアクションボタンを設置。
  * オフライン時の完了画面では、解説およびスコアの確認のみが可能となり、良問評価（👍/👎）、間違い指摘の送信、作家リアクション、SNS共有などの機能は無効（非活性化、オフライン警告メッセージ表示）となります。オンライン復帰後に未同期データがある場合は、『バックグラウンドで解答結果を同期中...』のインジケータが表示されます。
  * クイズに対する良問評価（👍良問 / 👎悪問）および体感難易度（1〜10の10段階評価、同一ユーザーは最新データで上書き）の匿名投票フォーム。面白さ星評価（星1〜5段階）は廃止し良問評価に統合。
  * **良問評価UI**: 全問解答完了したユーザーが「👍 良問」または「👎 悪問」を投票できるボタン（作成者本人は非表示）。悪問選択時は任意でフリーテキスト理由を入力可能。投票結果は `ReviewService.submitReview` で `quizReviews` コレクションに保存し、`positiveCount`/`negativeCount` を更新する。
  * **再評価期間中の表示**: クイズ作成者が「リセット」を申請し、7日間の仮リセット期間中にあるクイズの場合、画面上に「⚠️ 修正に伴う再評価期間中」の一時バッジが表示され、過去の評価点数や「要改善」バッジ表示は一時的にマスク（非表示）される。
  * **間違い・別解指摘ボタン**: クリックすると、クローズドな指摘フォーム（指摘カテゴリ「誤植・事実誤認・別解の提案」と指摘内容の記述）が起動。
  * **作家リアクション（いいね・感謝）**: プレイしてくれたことへの感謝やお礼として、作成者に直接「いいね・リアクション」を送信するボタン。
  * **SNSワンクリック共有**: X等へ「クイズタイトル」「正解数」「経過秒数」を含んだ短縮URL付きのプレフォーマット文で共有するボタン。
* **遷移ルール**:
  * 「もう一度プレイ」ボタンで「クイズプレイ画面」に再遷移。
  * 「指摘送信」完了時、ポップアップで感謝のダイアログを表示し元の画面に留まる。
  * 作者のアイコン・表示名クリックで「プロフィール画面」へ遷移。

### 2.6 クイズ作成・編集画面 (Page: `/quiz/create`, `/quiz/[id]/edit`)
* **機能概要**:
  * クイズおよび問題群をビジュアルに設計・編集・公開・削除できるクリエイター向けUI。
  * **レイアウト**: 画面全体は縦1カラム構成。上部に「クイズの基本設定」カードを配置し、タイトル・説明・カバー画像アップロード・初期難易度・ジャンル・タグをひとつのカード内に集約する（旧来のサイドバー分割レイアウトを廃止）。カード内はカバー画像エリア（左）とその他メタ情報（右）の2カラムサブグリッドで整理し、スマートフォン幅では1カラムに折り畳まれる。
    * **カバー画像アップロード**: SVG-based XSS脆弱性を防止するためMIMEタイプ `'image/svg+xml'` (SVG形式) のアップロードは完全に禁止され、PNG/JPEG/GIF形式のみが許容される。
    * **ジャンル入力**: セレクトボックスによる既存ジャンルマスタ（`metadata_genres`）からの**必須選択**（下書き保存・公開のいずれも未選択不可）。**デフォルト選択値は設定されず、未選択（プレースホルダー「ジャンルを選択してください」）状態からスタートする。** 目的のジャンルが存在しない場合は、入力UIの隣に「新しいジャンルを申請する」動線リンクを設け、申請用モーダルまたは別画面（モデレータ投票による自治承認フロー）へと誘導する。
    * **タグ入力**: 最大5個まで設定可能。ユーザーのテキスト入力時にフロントエンドで「自動名寄せ（Normalizer：トリム、小文字化、スペース・記号の排除）」をリアルタイム実行。正規化されたID（`id`）に基づき、既存マスタに類似タグ（例：`'react'` 等）が検知された場合、入力欄の下に **「推奨: 類似するタグ #React が既に存在します。新しいタグを作成する代わりに、既存のタグを使用することをお勧めします。」** という推奨サジェスト警告UIをインライン表示し、既存タグへの置き換えを強く促す。
  * **設問エディタ**: 設問を動的に追加・削除。各設問に**問題文（必須）**テキストエリアを配置（ラベル「問題文（必須）」、`minLength` 5 / `maxLength` 500、未入力時は赤枠と直下エラー）。各設問のタイプ（選択式 / 短答文字入力式 / 並び替え / 連想 / 水平思考等）をトグル切り替え。
    - **並び替えエディタ**: 並び替えるための2〜6個の要素テキストの入力フォームをソート可能リストとして表示。ドラッグハンドル付きの**ドラッグ＆ドロップ**でエディタ上の並び順（＝正解順）を直接設定し、ドロップのたびに上から `correctOrder` を 0 始まりの連番で自動再採番します（上下ボタンによる順序変更は使用しません）。推奨実装は `@dnd-kit/core`（クイズリスト編集の D&D と同一方針）。
    - **連想エディタ**: 解答に至るための段階的ヒントリスト（1〜5個）の追加・削除が可能な複数行テキストフォーム。および、プレイヤーの回答と照合するための「正解テキストパターン（表記揺れパターン対応のカンマ区切り）」の入力フォーム。
  * 選択肢数（**2択〜10択**）の動的追加・削除、正解フラグ設定（**多肢選択では正解となる選択肢に複数チェック可能**）。選択肢は「＋ 選択肢を追加する」ボタンで1件ずつ追加（最大10件）、各行のゴミ箱アイコンで削除（最低2件）。正解が1件もなくなる削除は行わず、最後の1件の正解チェックは外せない。正解選択肢を削除した場合は先頭の選択肢が自動的に正解に設定される。**記述式**では入力タイプ（通常／数値／文字数指定）をトグル切り替え。数値は整数・小数の正解候補、文字数指定は要求文字数（1〜100）と正解候補の文字数一致を公開時に検証。表記揺れ対策の複数正解リスト入力。水平思考形式（ウミガメのスープ）では、自動真相判定のための必須正解キーワード（エッセンス）をタグ形式で複数追加・削除できるUIを提供。
  * 各問題に対する参考画像アタッチ、ヒント設定、制限秒数設定、マークダウン対応解説入力、および**出典・参考URL入力**（`sourceUrl`：任意。Wikipedia や公式ドキュメント等の根拠URLを設問ごとに1件登録できる）。
  * **出題形式の切り替え（フォーマット変更）**: 画面上部のフォーマット選択で出題形式を変更する際、すべての設問が初期値のまま（問題文・選択肢テキスト・正解設定等が未入力）であれば確認ダイアログなしに即座に切り替える。いずれかの設問に入力済みの内容がある場合のみ「既存データが一括変換されます。よろしいですか？」の確認ダイアログを表示する。
  * **公開・下書き保存時バリデーション**: 「下書き保存」はタイトル・ジャンル必須に加え、**登録済み各設問の問題文必須**（`collectQuestionTextValidationErrors`：トリム後5文字以上・500文字以内）。「公開」申請時は `validateQuizForPublish` によりタイトル・**ジャンル**・各設問の**問題文**・正解設定等を厳格に検証する（問題文は Zod `questionSchema` と二重整合）。不整合があれば画面上部にエラーサマリーを表示し、**該当する各入力セルの直下にもエラーメッセージを表示**（赤枠強調付き、`questionField: 'questionText'` 含む）。最初のエラー箇所へ自動スクロール。水平思考クイズの場合は、必須正解キーワードが最低1つ以上登録されていることをバリデーションで強制する。
  * **テストプレイ（F-206）**: 画面下部アクションバーに「テストプレイ」ボタンを配置（「下書き保存」「公開する」と並列）。クリック時、エディタ上の現在値を `sessionStorage`（`quizeum_test_play_payload`）へシリアライズし、問題文が入力された設問が1問以上あれば `/quiz/test-play/play?mode=normal` へ遷移する。公開用バリデーション・NGワードチェック・下書き保存は不要。未保存の新規作成・編集中下書きのいずれでも利用可能。
* **遷移ルール**:
  * 「下書き保存」または「公開」成功後、「作家ダッシュボード」または「ホーム画面」へ遷移。
  * 「テストプレイ」クリック時、プレイ可能な設問が0件ならトースト警告で遷移をブロック。1問以上あれば「テストプレイ画面」へ遷移。
  * 削除実行時は「本当に削除しますか？」の確認を経て、削除成功後に「作家ダッシュボード」へ遷移。

### 2.6.1 テストプレイ画面 (Page: `/quiz/test-play/play`) [F-206]
* **機能概要**:
  * 作問エディタから渡された draft データを `sessionStorage` から読み込み、通常プレイ画面（2.4）と同一の解答 UI で動作確認する。
  * 画面上部に「🔬 テストプレイ中（統計には記録されません）」バナーを常時表示。
  * 模擬試験・フラッシュカードモードの切り替え UI は非表示（通常モード固定）。
  * 正解未設定・解説未入力など未完成設問は、正誤判定をスキップし入力内容と解説（あれば）のみ表示するフォールバックを適用。
  * 水平思考（ウミガメのスープ）設問では AI API を呼び出さず、AI 判定不可の案内を表示。`truthKeywords` が設定済みの場合のみローカル部分一致判定を提供。
  * 設問ブックマーク・マイリスト追加 UI は非表示。
* **遷移ルール**:
  * 全問解答完了後、「テストプレイ結果画面」へ自動遷移。
  * `sessionStorage` の payload が無効（未ログイン・TTL 超過・authorId 不一致）の場合、作問画面またはログイン画面へリダイレクト。
  * ブラウザバックで作問画面へ戻った場合、エディタ state は React 側で保持（未保存変更は失われない）。

### 2.6.2 テストプレイ結果画面 (Page: `/quiz/test-play/result`) [F-206]
* **機能概要**:
  * 正解数・総問題数・正解率・設問別正誤一覧・解説を表示（通常結果画面の簡易版）。
  * 良問評価・体感難易度・間違い指摘・リアクション・SNS 共有は非表示。
  * 「編集画面に戻る」ボタンを主要 CTA として配置。
* **遷移ルール**:
  * 「編集画面に戻る」で `sourcePath`（または `history.back()`）により作問画面へ復帰。復帰時に `sessionStorage` のテストプレイ関連キーを破棄。

### 2.7 作家ダッシュボード (Page: `/creator/dashboard`)
* **機能概要**:
  * 自身が投稿したクイズ一覧と、累計プレイ数、評価平均、ブックマーク数のアナリティクスサマリーをグラフで表示。
  * 各クイズ詳細分析: 各設問における「選択肢別・正解率別の解答割合（％）」を円グラフや棒グラフでビジュアル化。
  * **間違い指摘リスト**: プレイヤーからクローズドに送信された誤植・事実誤認指摘の一覧を一覧表示。作家は指摘された該当問題をワンクリックで修正画面に読み込め、「解決済」にマーク可能。
  * **クイズ一括エクスポート**: 自身が作成したクイズ（下書き・公開中を含む）をまとめてJSON形式でダウンロード（出力）できる「一括エクスポート」ボタンを設置。バックアップや配布用パッケージデータとして出力可能です。※一括インポート機能は廃止されたため、JSONファイルのインポートエリアは設置されません。
* **遷移ルール**:
  * クイズの新規作成、または指摘の「修正する」ボタンクリックにより、該当クイズの「クイズ編集画面」へ遷移。
  * 指摘解決完了時、対象のプレイヤーへ「修正完了感謝通知」が自動トリガー（バックグラウンド通知）され、ダッシュボード上に完了ステータスが反映される。

### 2.8 プロフィール画面 (Page: `/profile/[uid]`)
* **機能概要**:
  * ユーザーの基本情報（アバター、表示名、自己紹介、フォロー/フォロワー数）を表示。
  * 獲得した「称号バッジ一覧」をアイコンバッジ付きで表示。
  * **信頼スコアと権限ティアー**: `reputationScore`（例: `283 pt`）と現在の `moderationTier` に対応する色付きティアーバッジ（Newcomer/Contributor/Moderator/Senior Moderator）を表示する。
  * 自身が作成した公開クイズ一覧、および作成したクイズリスト一覧をタブで切り替えて表示。
  * **ソーシャルリンク**: フォロー数・フォロワー数エリアをクリック可能にし、接続者一覧を表示。
  * **弱点克服セクション**: 未復習の間違い問題が存在する場合（`totalFailedQuestionsCount > 0`）、その問題数と「弱点克服プレイ（復習）」を開始するボタンを表示。
  * **退会処理中のフォールバック表示**: 対象ユーザーの `deleteStatus` が `'delete_pending'`（退会処理中）の場合、画面上に「このアカウントは削除処理中です」とフォールバックメッセージを表示し、非公開情報や作成リスト等の表示エリアを一時非活性化または非表示にします。
* **遷移ルール**:
  * 自身のプロフィールの場合は「プロフィール編集画面」への遷移ボタンを表示。
  * 他人のプロフィールの場合は「フォロー/フォロー解除」のアトミックなトグルボタンを表示。
  * フォロー数/フォロワー数エリア of クリックで「フォロー/フォロワー一覧画面」へ遷移。
  * リアクション履歴リンクのクリックで「リアクション履歴画面」へ遷移。
  * 「復習プレイを開始」ボタンクリックにより、「弱点克服プレイ画面」へ遷移。
  * **退会過渡状態アクセス制限**: `deleteStatus == 'delete_pending'` のプロフィール画面に直接アクセスした場合は、404画面へとフォールバックされ遷移がブロックされます。

* **機能概要**:

  * リストのタイトル、詳細説明、作成者プロフィール、および収録されているクイズの一覧カードを表示。

### 2.10 リスト作成・編集画面 (Page: `/list/create`, `/list/[id]/edit`)
* **機能概要**:

  * コンテンツ選択UI（クイズ＆設問対応）: 自身が投稿したクイズ、お気に入りのクイズ、またはブックマークした設問からキーワード検索やタグ検索し、リストに要素（クイズまたは設問単体）をアタッチ/デタッチ（順序のドラッグ＆ドロップ並び替え対応）。

### 2.11 弱点克服（復習プレイ）画面 (Page: `/quiz/review`)
* **機能概要**:
  * ログイン中ユーザーが過去に間違えた問題（`attempts` コレクション内の `failedQuestionIds` に記録されている設問）をバックエンドから一括フェッチして構成される特別なプレイ画面。
  * **ジャンル選択UI**: プレイ開始前に出題対象のジャンルを選択できる（「オールジャンル」選択も可能）。選択されたジャンルに属するクイズの間違い問題のみが出題対象となる。`AttemptService.getFailedQuestions(userId, undefined, genreFilter)` で取得。
  * ユーザーは「間違えた問題のみ」で構成された即席クイズセッションに挑戦し、通常プレイと同様に回答・正誤判定を受ける。
  * 正解した問題は、復習プレイ完了（結果画面）時に一括バッチ処理で `AttemptService.updateFailedQuestions` を呼び出してリストからまとめて削除され、`users.totalFailedQuestionsCount` もアトミックに減算される。
* **遷移ルール**:
  * すべての復習対象問題を解き終えた後、「クイズ結果画面」へ遷移。

  * ジャンルの紹介、およびそのジャンルに特化したクイズ群を「トレンド順」「人気順」「新着順」でソート可能。
  * `metadata_genres` のメタデータ（`mergedGenreIds`）を参照し、マージされた旧ジャンル名のクイズも同一ジャンルとして一覧表示する（仮想統合・シノニム展開）。
* **遷移ルール**:
  * クイズカードをクリックすると、「クイズ詳細画面」へ遷移。

### 2.20 タグ/ジャンルマージリクエスト画面 (Page: `/community/merge`)
* **機能概要**:
  * コミュニティモデレータがタグやジャンルの統廃合（マージ）を提案・投票する画面。
  * **起案タブ**: `moderationTier >= 'moderator'` のユーザーが `TagMergeService.createMergeRequest` を通じて新規マージ提案を起案できる。
  * **投票タブ**: 現在 `status = 'pending'` のマージリクエスト一覧を表示。各リクエストに対して「賛成 👍 / 反対 👎」を投票可能（モデレータ資格ユーザーのみ）。`weightedVotesFor`/`weightedVotesAgainst` と賛成率のプログレスバーをリアルタイム表示。
  * **投票重み表示**: 自分の `moderationTier` が `senior_moderator` の場合は「投票重み: x2」と表示。
* **遷移ルール**:
  * リクエストのソースタグ/ジャンルをクリックすると、対応する「タグ別クイズ一覧」または「ジャンル別クイズ一覧」へ遷移（内容確認用）。

### 2.21 ジャンル新設申請・投票画面 (Page: `/community/genres`)
* **機能概要**:
  * 認証済みユーザーが新しいジャンルの追加を申請し、モデレータが投票で承認・否決する画面。
  * **申請フォーム**: ジャンルID（英小文字・ハイフン区切り）、日本語表示名、およびジャンルを象徴するアイコン画像（PNG/JPEG/GIF形式、SVGは不可）を Firebase Storage にアップロードして登録。SVG-based XSS脆弱性を防止するため、MIMEタイプ `'image/svg+xml'` (SVG形式) のアップロードは完全に禁止されている。`TagMergeService.submitGenreRequest` を使用。
  * **投票タブ**: `status = 'pending'` のジャンル新設申請一覧を表示。モデレータ資格ユーザーが賛成/反対投票を行い、可決条件（重み付き賛成票 ≥ 5 かつ賛成率 ≥ 80%）を満たした時点でシステムが自動的に `metadata_genres` にドキュメントを作成し、全画面に即時反映。
* **遷移ルール**:
  * 可決・否決されたリクエストは「承認済み / 否決済み」タブに移動し、履歴として閲覧可能。
