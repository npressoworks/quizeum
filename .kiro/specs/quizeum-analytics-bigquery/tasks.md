# Implementation Plan

## 1. 土台構築: データモデルと型の拡張
- [x] 1.1 (P) 詳細解答行動ログ用の型定義の追加
  - すべての問題形式（真偽値、多肢選択、記述、早押し、並べ替え、連想、水平思考）に対応する `QuestionAnswerDetail` 型を追加
  - 既存のクイズ試行記録である `Attempt` インターフェースを拡張し、詳細配列を受け取れるようにする
  - 型定義の追加後にビルドがエラーなく通ることを確認
  - _Requirements: 1.5_
  - _Boundary: Types_
- [x] 1.2 (P) プレイ進行進捗およびオフライン同期用の型拡張
  - プレイ中進捗データ `PlayProgressData` およびオフライン退避データ `PendingSyncAttempt` に `QuestionAnswerDetail` の配列を追加
  - シリアライズ・デシリアライズ処理が拡張された詳細解答情報を保持することを確認
  - 進捗データ構造の更新が既存のセッションマネージャと互換性を維持していることを確認
  - _Requirements: 3.1_
  - _Boundary: AttemptSession_

## 2. コア機能: トラッキングロジックと保存サービスの拡張
- [x] 2.1 クイズプレイ状態管理フックへのトラッキング追加
  - クイズプレイ状態管理フック内に設問ごとの解答開始時間およびアクション（ヒント表示、回答変更等）を監視・蓄積するロジックを実装
  - 設問が決定された瞬間に、該当する設問形式特有のプロパティを抽出し詳細解答オブジェクトを構築して配列に格納する
  - 解答時間の測定処理が一時停止やカウントダウン処理と干渉しないことを単体テストで確認
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 2.4_
  - _Boundary: PlayStateHook_
  - _Depends: 1.1, 1.2_
- [x] 2.2 (P) 水平思考クイズ (ウミガメスープ) AI 判定 API のデータ保存拡張
  - AI真相判定 API 側で、ユーザーの真相提出（合格・あきらめ完了）のタイミングで詳細データオブジェクトを作成する処理を実装
  - 質問数、最終真相解答文字列、合格/ギブアップのステータスをアトミックに Firestore へ書き込む
  - テストクライアントから API を呼び出した際、Firestore の該当ドキュメントに期待通りの詳細データが書き込まれることを確認
  - _Requirements: 2.5_
  - _Boundary: VerifyTruthAPI_
  - _Depends: 1.1_
- [x] 2.3 プレイ結果保存処理とオフライン同期の拡張
  - プレイ結果保存サービス側で詳細データ配列を受け取り、問題数や正解数のチート防止用二重検証を追加
  - ネットワークオンライン復旧時におけるバッチ同期処理で、詳細データを含んだ attempt レコードを Firestore に同期送信できることを確認
  - 同期中にエラーが発生した場合、ローカルストレージの未同期データが破棄されずに復元されることを確認
  - _Requirements: 1.5, 3.2, 3.3_
  - _Boundary: AttemptService_
  - _Depends: 2.1_
- [x] 2.4 (P) BigQuery 同期スキーマおよび移行ガイドの配置
  - 同期先 BigQuery テーブル用のカラムスキーマ定義 JSON を作成
  - 配列データをアンネスト展開して設問ごとのフラットな統計行にする分析用ビューの SQL スクリプトを作成
  - 既存の Firestore 内の履歴データを BigQuery に一括移行するためのインポートコマンド手順書をドキュメントとして作成
  - _Requirements: 4.1, 4.2, 4.3_
  - _Boundary: BigQueryAnalytics_
